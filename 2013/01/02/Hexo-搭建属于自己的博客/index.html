基于TravisCI实现hexo博客持续继承

一、Travis CI

  1. 持续集成CI(Continuous Integration)，一种自动编译和发布代码的技术。
  2. Travis CI是在线托管的CI服务，用Travis来进行持续集成，不需要自己搭服务器，在网页上点几下就好，用起来更方便。它对开源项目是免费的，并且与Github实现了无缝衔接。

三、开始实现(https://www.travis-ci.com/)

hexo deploy报错

一、问题

  1. hexo clean正常
  2. hexo generate正常
  3. hexo deploy最后一步报错
  • 搜索hexo deploy步骤，发现在_config.yaml有一个deploy配置，这是之前自己电脑配两个github账户修改的，换成正常的github就行了
  1. 更换后再次尝试：hexo deploy，报错
  • 设置代理，具体可以看梯子监听的端口号
    • git config –global http.proxy http://127.0.0.1:7890
    • git config –global https.proxy http://127.0.0.1:7890
  1. 再次尝试：hexo deploy，报错
  • 访问报错给的链接：https://github.com/liusirdotme/liusirdotme.github.io/security/secret-scanning/unblock-secret/2vqtYDF0HZx3n8Uw1qjTjHz013v，允许访问
  1. 再次尝试：hexo deploy，报错
  • 多次重试，最终成功

访问报错

一、问题

二、解决

  1. 网上搜了各种处理办法，都不行
  2. 去域名服务商后台查看，多了个A解析，貌似是服务商自动加的(Godaday)
  3. 删除无用的A解析，恢复正常
